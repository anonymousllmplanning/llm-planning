# Optional Improvement Source Alignment

This directory is generated by `scripts/analysis/compute_optional_improvements.py`.
The diagnostics use only paper result artifacts, task ids, model names, retained-ordering counts, and metric columns. They do not include raw GAIA questions, answers, or attachments.

## Inputs used

- Open-weight main pool: `version2/paper_results/official_modified_gt_20260504/gaia_per_sample_official_modified_gt.csv`
  - 7 models x 165 tasks = 1,155 rows.
  - This is the post-processed per-sample artifact aligned to the main paper's seven open-weight models.
- Closed-model extension:
  - `version2/paper_results/openai_analysis/openai_combined_per_sample_stage3fb.csv`
  - `version2/paper_results/gemini_analysis/gemini_combined_per_sample.csv`
  - 8 models x 165 tasks = 1,320 rows.
- DAG and retained-ordering profile:
  - `data/Augmented/DAGs/gaia_cat_A_async_plan.jsonl`
  - `data/Augmented/DAGs/gaia_cat_B_async_plan.jsonl`
  - `data/Augmented/DAGs/gaia_cat_C_async_plan.jsonl`
  - `data/Augmented/DAGs/gaia_cat_D_async_plan.jsonl`
  - 165 task-level DAG records.

## Alignment note

The raw per-category result files under `/tmp/thchuang/llm_planning_work/LLM-planning/organized_results/gaia` are useful for archival tracing, but the optional-improvement diagnostics are computed from the final post-processed paper artifacts listed above. This keeps the numbers aligned with the main paper tables after modified-GT handling, Stage-3 fallback handling, human answer audit updates, and parameter normalization.
